# 니어립

니어립은 원하는 책을 검색하고, 가까운 도서관의 소장 여부와 대출 가능 여부를 확인할 수 있는 서비스입니다.

## 소개

니어립은 `0.1.0` 기준으로 정리한 MVP입니다. 사용자는 책을 검색한 뒤, 해당 책을 가진 근처 도서관을 찾고 지금 바로 빌릴 수 있는지까지 한 번에 확인할 수 있습니다. 저장소는 React 기반 web, Fastify 기반 BFF, 공통 타입 패키지로 구성된 `pnpm` workspace 모노레포입니다.

개발 기간은 `2026.03.31 ~ 2026.04.17`입니다.

## 서비스 주소

- web: `https://nearlib.com`
- bff: `https://api.nearlib.com`

## 서비스가 해결하는 문제

책을 매번 사기 어려운 사람에게 도서관은 좋은 대안이지만, 원하는 책을 가진 도서관이 근처에 있는지와 지금 대출 가능한 상태인지를 빠르게 확인하기는 쉽지 않습니다. 니어립은 이 과정을 줄여, 검색부터 도서관 확인과 대출 가능 여부 확인까지 한 서비스 안에서 이어지게 만드는 데 집중합니다.

## 핵심 기능

- 책 제목이나 저자명으로 원하는 도서를 찾을 수 있습니다.
- 검색 결과를 다시 불러오거나 다른 조건으로 바로 다시 검색할 수 있습니다.
- 책 상세 정보와 대출 관련 정보를 함께 확인할 수 있습니다.
- 선택한 지역 안에서 해당 책을 가진 도서관을 찾을 수 있습니다.
- 도서관 목록과 지도를 함께 보며 위치와 기본 정보를 비교할 수 있습니다.
- 특정 도서관 기준으로 지금 대출 가능한 상태인지 확인할 수 있습니다.

## 주요 사용자 플로우

### 1. 읽고 싶은 책이 이미 정해져 있을 때

사용자는 이미 책 제목이나 저자명을 알고 있기 때문에 홈에서 바로 검색을 시작합니다. `/books` 결과 화면에서 원하는 책을 고른 뒤 `소장 도서관 찾기`를 눌러, 이 책을 가진 근처 도서관이 어디 있는지 확인합니다. 이 경우의 핵심은 책을 다시 비교하는 것이 아니라, 가능한 한 빨리 도서관 후보를 찾는 것입니다.

### 2. 검색 결과가 여러 권이라 어떤 책인지 다시 확인해야 할 때

같은 제목의 다른 책이 함께 나오거나, 찾는 판본이 맞는지 바로 확신하기 어려울 수 있습니다. 이때 사용자는 `/books` 결과 화면에서 후보를 비교하고, 필요하면 `상세 보기`를 열어 저자, 출판 정보, 대출 관련 정보를 더 확인합니다. 찾는 책이 맞다고 판단한 뒤에만 `소장 도서관 찾기`로 넘어가므로, 이 플로우의 핵심은 도서관을 찾기 전에 책을 정확히 고르는 것입니다.

### 3. 도서관은 찾았지만 지금 바로 빌릴 수 있는지가 중요할 때

책을 가진 도서관이 여러 곳일 수 있기 때문에, 사용자는 지역을 고른 뒤 도서관 목록과 지도를 함께 보며 어디를 확인할지 좁혀 갑니다. 이후 특정 도서관을 선택하고 `대출 가능 여부 조회`를 눌러, 단순히 책이 있는지뿐 아니라 지금 바로 빌릴 수 있는 상태인지까지 확인합니다. 이 플로우의 핵심은 가까운 도서관을 찾는 것에서 끝나지 않고, 실제로 바로 이용할 수 있는 곳을 골라내는 것입니다.

## 아키텍처

니어립은 `pnpm` workspace 기반 모노레포로 구성되어 있습니다. 사용자가 보는 화면, 외부 도서 Open API를 다루는 서버, 그리고 두 쪽이 함께 쓰는 타입을 나눠 관리합니다.

- `apps/web`
  - 사용자가 직접 보는 화면과 상호작용을 담당합니다.
  - 홈 검색, 검색 결과, 책 상세 보기, 지역 선택, 도서관 결과, 대출 가능 여부 확인 화면을 포함합니다.
- `apps/bff`
  - 외부 도서 Open API를 대신 호출하고, 화면에서 바로 쓰기 좋은 형태로 값을 다시 정리합니다.
  - 외부 API 인증키는 BFF 환경 변수에서만 관리합니다.
- `packages/contracts`
  - web과 bff가 함께 쓰는 공통 타입을 담습니다.
  - 화면과 서버가 같은 요청 값과 응답 값을 보게 합니다.

이 구조를 택한 이유는 화면 코드와 외부 API 처리를 분리하고, 외부 도서 Open API 인증키를 브라우저에 두지 않으며, web과 bff가 같은 데이터 형식을 기준으로 동작하게 만들기 위해서입니다.

## 기술 선택 이유

### React

니어립은 검색 시작, 검색 결과 확인, 책 상세 보기, 지역 선택, 도서관 결과 확인처럼 사용자가 한 번 들어온 뒤 여러 화면과 다이얼로그를 빠르게 오가며 확인하는 서비스입니다. 이 프로젝트에서는 검색 엔진 최적화보다, 한 번 앱이 로딩된 뒤의 매끄러운 화면 전환과 상호작용이 더 중요했습니다. 그래서 서버에서 HTML을 먼저 만드는 구조보다, 클라이언트에서 빠르게 반응하는 React 기반 구성이 더 잘 맞았습니다.

### Fastify BFF

니어립은 외부 도서 Open API를 브라우저에서 직접 호출하지 않습니다. 가장 큰 이유는 인증키를 브라우저에 두지 않기 위해서입니다. 그래서 web 앞단에 BFF를 두고, 외부 API 호출과 응답 정리를 서버에서 맡기도록 했습니다.

Fastify를 선택한 이유는 지금 필요한 역할에 잘 맞기 때문입니다. 니어립의 BFF는 무거운 서버 애플리케이션보다는, 외부 도서 Open API를 대신 호출하고 응답을 서비스에 맞게 다시 정리하는 얇은 API 서버에 가깝습니다. Fastify는 이런 구조를 단순하게 유지하기 좋고, route 단위 구성이 명확하며, 지금처럼 통합테스트를 서버 포트 없이 `inject()`로 검증하기에도 편합니다. 이후 로그인, 북마크, 추천, 데이터 저장 기능이 붙더라도 같은 서버 안에서 확장하기 쉽다는 점도 같이 봤습니다.

### contracts 패키지

web과 bff는 같은 요청 값과 응답 값을 봐야 합니다. 그렇지 않으면 화면은 어떤 값을 기대하고, 서버는 다른 값을 내려주는 문제가 자주 생깁니다. 그래서 `packages/contracts`를 따로 두고, 두 쪽이 같은 타입을 기준으로 움직이게 했습니다.

이 구조 덕분에 화면은 외부 도서 Open API의 복잡한 원본 응답을 직접 알 필요가 없고, BFF는 서비스에 필요한 값만 다시 정리해 안정적으로 내려줄 수 있습니다.

## 요구 환경

- Node.js 24 이상
- pnpm 10 이상

현재 저장소는 Node `24.12.0`, pnpm `10.28.0` 환경에서 확인했습니다.

## 빠른 시작

1. 의존성을 설치합니다.

```bash
pnpm install
```

2. 예시 파일을 기준으로 환경 변수를 준비합니다.

```bash
cp apps/web/.env.example apps/web/.env
cp apps/bff/.env.example apps/bff/.env
```

3. 아래 값을 채웁니다.

- `apps/web/.env`
  - `VITE_API_BASE_URL`: BFF 주소
  - `VITE_KAKAO_MAP_APP_KEY`: 카카오 지도 키
- `apps/bff/.env`
  - `WEB_APP_ORIGIN`: 운영 기준 web origin
  - `LIBRARY_API_BASE_URL`: 외부 도서 Open API 주소
  - `LIBRARY_API_AUTH_KEY`: 외부 도서 Open API 인증키

로컬 개발에서는 `apps/bff/.env`의 `ALLOW_DEV_CORS_ORIGINS=true`로 두면 `http://localhost:5173`, `http://127.0.0.1:5173` 요청을 임시로 허용할 수 있습니다.

4. web과 bff를 함께 실행합니다.

```bash
pnpm dev:all
```

- web: `http://127.0.0.1:5173`
- bff: `http://127.0.0.1:3001`

## 주요 스크립트

- `pnpm dev`
  - web만 실행합니다.
- `pnpm dev:bff`
  - bff만 실행합니다.
- `pnpm dev:all`
  - web과 bff를 함께 실행합니다.
- `pnpm build`
  - web build를 실행합니다.
- `pnpm build:bff`
  - bff build를 실행합니다.
- `pnpm build:all`
  - workspace 전체 build를 실행합니다.
- `pnpm typecheck:all`
  - workspace 전체 타입 점검을 실행합니다.
- `pnpm test:web`
  - web 테스트를 watch 모드로 실행합니다.
- `pnpm test:run:web`
  - web 테스트를 1회 실행합니다.

## 환경 변수

- web 예시: `apps/web/.env.example`
- bff 예시: `apps/bff/.env.example`

주의할 점:

- web의 `VITE_API_BASE_URL`은 외부 API가 아니라 BFF를 가리켜야 합니다.
- 외부 도서 Open API 인증키는 BFF 환경 변수에서만 관리합니다.
- 카카오 지도 키는 web 환경 변수에서 설정합니다.
- `WEB_APP_ORIGIN`은 `https://` 기반의 정확한 origin 값이어야 합니다.
