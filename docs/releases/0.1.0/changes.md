# 0.1.0 Changes

이 문서는 `0.1.0`에 실제로 담긴 내용을 기능과 기술 기반 기준으로 정리한 릴리즈 문서입니다. 세부 수정 이력이나 커밋 목록을 나열하는 대신, 지금 MVP에서 사용자가 무엇을 경험할 수 있고 그 경험을 가능하게 하기 위해 어떤 기반을 정리했는지에 집중합니다.

## 0.1.0에 담긴 내용

`0.1.0`은 니어립의 첫 MVP 릴리즈 스냅샷입니다. 사용자가 원하는 책을 검색하고, 그 책을 소장한 가까운 도서관을 찾고, 지금 바로 빌릴 수 있는지까지 확인하는 흐름이 하나의 서비스 경험으로 이어지도록 정리했습니다.

## 현재 구현된 사용자가 경험 가능한 기능

- 홈에서 책 제목이나 저자명으로 검색을 시작할 수 있습니다.
- `/books` 결과 화면에서 검색 결과를 확인하고, 같은 화면 안에서 다시 검색하거나 페이지를 이동할 수 있습니다.
- 각 결과 카드에서 `상세 보기`를 눌러 책의 기본 정보와 대출 관련 정보를 확인할 수 있습니다.
- `소장 도서관 찾기`를 눌러 지역을 고르고, 해당 책을 가진 도서관을 바로 찾을 수 있습니다.
- 도서관 결과 다이얼로그에서 리스트와 지도를 함께 보며 도서관 위치와 정보를 비교할 수 있습니다.
- 특정 도서관을 선택한 뒤 `대출 가능 여부 조회`를 눌러 지금 바로 빌릴 수 있는지 확인할 수 있습니다.
- 검색 결과는 URL을 기준으로 복원되므로, 새로고침이나 직접 진입, 뒤로가기 이후에도 같은 결과 흐름을 다시 이어갈 수 있습니다.

## 이 기능을 가능하게 한 기술적 기반

- `web`, `bff`, `contracts`의 역할을 나눠 서비스 경계를 정리했습니다.
  - 사용자가 보는 화면은 `apps/web`, 외부 도서 Open API 호출과 응답 정리는 `apps/bff`, 공통 요청/응답 기준은 `packages/contracts`가 맡습니다.
- BFF를 두어 외부 도서 Open API 인증키를 브라우저에 두지 않는 구조를 만들었습니다.
  - web은 외부 API를 직접 호출하지 않고 BFF만 호출합니다.
  - BFF는 외부 응답을 그대로 넘기지 않고, 화면에서 바로 쓰기 쉬운 내부 계약으로 정규화합니다.
- `/books`를 중심으로 검색 결과와 후속 흐름을 정리했습니다.
  - 검색 결과는 URL 기반으로 복원되고, 책 상세 보기·지역 선택·도서관 결과가 같은 문맥 안에서 이어지도록 만들었습니다.
- 책 상세, 지역 선택, 도서관 결과, 대출 가능 여부 조회까지 각 단계의 계약을 분리해 정리했습니다.
  - 덕분에 각 기능은 서로 연결되지만, 입력과 결과 기준은 비교적 명확하게 유지됩니다.
- route와 dialog 기준의 lazy loading을 적용해 초기 번들 비용을 줄였습니다.
  - `/books` 결과 문맥과 상세/지역/도서관 결과 dialog는 필요 시점에만 로드되도록 정리했습니다.
  - 그 결과 web build 기준 chunk size warning 없이 현재 번들 구성을 유지할 수 있게 됐습니다.

## 이번 버전에 포함되지 않은 것

- 계정, 로그인, 개인화 기능은 아직 포함하지 않았습니다.
- 북마크, 예약, 알림 같은 후속 사용자 기능도 아직 없습니다.
- 배포 자동화, 릴리즈 자동화, 운영 자동화는 이번 버전 범위에 포함하지 않았습니다.
- MVP 이후 품질 개선 후보는 남아 있지만, 이번 문서에서는 현재 사용 가능한 흐름을 먼저 고정하는 데 집중했습니다.
