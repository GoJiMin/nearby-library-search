# 0.1.0 Verification

이 문서는 `0.1.0`을 배포 가능한 상태로 판단할 때 사용한 근거를 정리한 기록입니다. 목적은 “대충 확인했다”를 적는 것이 아니라, **무엇을 검증했고, 어떤 방식으로 확인했고, 결과가 어땠는지**를 다음 개발자가 그대로 따라갈 수 있게 남기는 데 있습니다.

## 배포 가능하다고 판단한 이유

`0.1.0`은 아래 네 가지를 통과했기 때문에 배포 가능한 상태로 판단했습니다.

- 프로젝트 내 전체 타입 점검과 웹과 서버가 정상적으로 빌드됩니다.
- 웹 코드베이스 전체 테스트와, 그 안에 포함된 핵심 통합테스트가 통과했습니다.
- 서버 코드베이스 전체 테스트와, 그 안에 포함된 핵심 라우트 통합테스트가 통과했습니다.

즉 `0.1.0`은 “기능이 있는 것처럼 보이는 상태”가 아니라, 검색부터 대출 가능 여부 확인까지의 핵심 기능에 대해 타입 점검, 빌드, 전체 테스트가 통과한 상태입니다.

## 무엇을 어떻게 검증했는가

### 1. 코드 기준과 배포 산출물

아래 세 가지 명령어를 실행했습니다.

- `pnpm exec tsc -b tsconfig.json`
- `pnpm --filter @nearby-library-search/web build`
- `pnpm --filter @nearby-library-search/bff build`

이 세 항목은 각각 아래를 확인하기 위한 명령어입니다.

- 타입 점검: web, bff, contracts가 같은 타입 정의를 기준으로 컴파일되는지 확인
- web build: 현재 화면 코드가 실제 배포 산출물로 다시 만들어지는지 확인
- bff build: 현재 서버 코드가 실제 배포 산출물로 다시 만들어지는지 확인

결과는 세 항목 모두 통과였습니다. 즉 “지금 저장소 상태로는 타입 단계와 배포 산출물 생성 단계에서 막히지 않는다”는 점이 다시 확인됐습니다.

### 2. web 사용자 시나리오

web에서는 일부 테스트만 따로 실행한 것이 아니라, 현재 구현된 코드베이스 전체 테스트를 다시 실행했습니다. 이번 확인은 커러비지 측정과 전체 테스트를 한 번 더 돌리는 방식으로 진행했습니다.

실행 명령은 아래와 같습니다.

```bash
pnpm --filter @nearby-library-search/web test:coverage
```

전체 실행 안에는 핵심 기능을 검증하기 위해 작성한 통합테스트도 포함됩니다. 대표적으로 아래 테스트들이 이번 전체 실행 안에 포함되어 통과했습니다.

- `apps/web/src/app/router/router.integration.test.tsx`
- `apps/web/src/features/book-search/ui/search-start/test/BookSearchStart.test.tsx`
- `apps/web/src/features/book-search/ui/search-result/test/BookSearchResult.test.tsx`
- `apps/web/src/features/region/ui/test/RegionSelectDialog.test.tsx`
- `apps/web/src/features/library/ui/test/LibrarySearchResultDialog.test.tsx`

이 통합테스트들은 아래 사용 시나리오를 검증하기 위해 만들어졌고, 이번 전체 테스트 실행에서도 함께 통과했습니다.

- 홈에서 책 제목 또는 저자명으로 검색을 시작할 수 있는지
- `/books` 주소가 결과 화면으로 다시 이어지는지
- 결과 화면에서 재검색과 페이지 이동이 되는지
- `상세 보기`가 열리고 관련 정보가 보이는지
- `소장 도서관 찾기` 이후 지역 선택이 이어지는지
- 도서관 결과 다이얼로그가 다시 열리고, 대출 가능 여부 조회로 이어질 수 있는지

전체 실행 결과는 아래와 같습니다.

- 테스트 파일 28개 통과
- 테스트 234개 통과

즉 현재 구현된 web 코드베이스 전체 테스트가 통과했고, 그 안에 포함된 핵심 통합테스트도 함께 통과했기 때문에 사용자가 실제로 경험하는 핵심 기능이 정상적으로 동작한다는 근거를 남길 수 있습니다.

### 3. bff 서버 시나리오

bff에서도 일부 라우트 테스트만 따로 실행한 것이 아니라, 현재 구현된 코드베이스 전체 테스트를 다시 실행했습니다. 이번 확인은 커버리지 측정을 겸해 전체 테스트를 한 번 더 돌리는 방식으로 진행했습니다.

실행 명령은 아래와 같습니다.

```bash
pnpm --filter @nearby-library-search/bff test:coverage
```

전체 실행 안에는 핵심 기능을 검증하기 위해 작성한 라우트 통합테스트도 포함됩니다. 대표적으로 아래 테스트들이 이번 전체 실행 안에 포함되어 통과했습니다.

- `apps/bff/src/routes/book/search/test/route.test.ts`
- `apps/bff/src/routes/book/detail/test/route.test.ts`
- `apps/bff/src/routes/library/search/test/route.test.ts`
- `apps/bff/src/routes/library/availability/test/route.test.ts`

이 통합테스트들은 아래 서버 동작을 검증하기 위해 만들어졌고, 이번 전체 테스트 실행에서도 함께 통과했습니다.

- 책 검색 요청이 정상 요청과 잘못된 요청을 구분하는지
- 책 상세 요청이 필요한 값을 다시 정리해 응답하는지
- 도서관 검색과 대출 가능 여부 조회가 지금 정의한 응답 형식대로 내려오는지
- 외부 도서 Open API가 빈 응답, 잘못된 응답, 실패 응답을 줄 때도 bff가 현재 기준의 오류 응답으로 다시 바꾸는지

전체 실행 결과는 아래와 같습니다.

- 테스트 파일 22개 통과
- 테스트 114개 통과

즉 현재 구현된 bff 코드베이스 전체 테스트가 통과했고, 그 안에 포함된 핵심 라우트 통합테스트도 함께 통과했기 때문에 서버가 정상 응답뿐 아니라 외부 API의 이상한 응답까지 현재 기준으로 정리하고 있다는 근거를 남길 수 있습니다.

## 운영에서 계속 볼 항목

이번 확인에서 바로 배포를 막을 정도의 문제는 없었지만, 이후 버전에서 계속 봐야 할 항목은 있습니다.

- 이 서비스는 외부 도서 Open API 응답 품질에 계속 영향을 받습니다.
  - 그래서 bff 검증에서도 정상 응답뿐 아니라 이상한 응답을 현재 기준 오류로 바꾸는지를 중요하게 확인했습니다.
- 브라우저 기반 E2E 자동화는 아직 도입하지 않았습니다.
  - 현재 문서의 근거는 타입체크, build, 전체 테스트, coverage 기준선에 기반합니다.

## 최종 판단

`0.1.0`은 지금 기준에서 배포 가능한 상태입니다. 판단 근거는 아래와 같습니다.

- 타입 점검과 배포 산출물 생성이 다시 통과했습니다.
- 현재 구현된 web 코드베이스 전체 테스트와 핵심 통합테스트가 통과했습니다.
- 현재 구현된 bff 코드베이스 전체 테스트와 핵심 route 통합테스트가 통과했습니다.

즉 `0.1.0`은 **검색부터 대출 가능 여부 확인이라는 핵심 기능이 동작함이 검증된 MVP**입니다.
