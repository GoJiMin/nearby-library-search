# 0.1.0 Known Issues

이 문서는 `0.1.0`에서 아직 남아 있는 점을 정리한 기록입니다. 목적은 지금 바로 사용을 막는 문제가 있는지, 아직 제공하지 않는 기능이 무엇인지, 외부 서비스에 기대는 부분에서 어떤 제약이 있는지를 새로 들어온 개발자가 바로 이해하게 만드는 데 있습니다.

## 0.1.0에서 아직 남아 있는 점

`0.1.0`은 검색부터 대출 가능 여부 확인까지의 핵심 기능은 사용할 수 있지만, 아직 첫 버전이라서 비어 있는 부분과 외부 서비스에 기대는 부분이 남아 있습니다. 이 문서는 그 범위를 서비스 관점에서 정리합니다.

## 현재 사용을 막는 문제

문서 작성 시점에 `0.1.0` 배포를 멈춰야 할 문제는 확인하지 못했습니다. 타입 점검, web과 bff build, 전체 테스트가 통과했고, 현재 문서 기준에서는 검색, 도서 상세 보기, 지역 선택, 도서관 검색, 대출 가능 여부 확인까지 이어지는 핵심 기능을 출시 불가능 상태로 만드는 문제는 남아 있지 않습니다.

## 현재 남아 있는 일반 한계

- 로그인, 개인화, 북마크, 예약, 알림 기능이 아직 없습니다.
- 검색 결과나 도서관 검색 결과를 저장해 두었다가 나중에 다시 꺼내 보는 기능이 없습니다.
- 지도는 도서관을 비교할 때 도움이 되지만, 지금 버전의 중심은 여전히 검색 결과와 도서관 목록 확인입니다.

## 외부 의존성으로 인한 제약

- 책 검색, 도서 상세, 도서관 검색, 대출 가능 여부 확인은 모두 외부 도서 Open API 응답에 영향을 받습니다. 외부 응답이 느리거나 비어 있거나 형태가 예상과 다르면, 니어립이 안내 메시지로 정리하더라도 결과 자체를 보장할 수는 없습니다.
- 일부 검색어 조합은 외부 도서 Open API가 정상 결과 대신 비어 있는 응답이나 이상한 응답을 줄 수 있습니다. 이런 경우 니어립은 다시 검색해 보라는 안내를 보여주지만, 사용자가 원하는 결과를 항상 돌려줄 수는 없습니다.
- 지도 표시는 카카오 지도 설정과 도서관 위치 정보가 함께 있어야 합니다. 둘 중 하나라도 부족하면 지도 대신 목록과 상세 정보 중심으로 확인해야 합니다.

## 다음 버전 후보

- 브라우저에서 실제 사용 과정을 따라가는 자동 테스트를 도입합니다.
- 자주 찾는 책이나 검색 결과를 저장해 두는 기능을 검토합니다.
- 검색 결과나 도서관 검색 결과를 나중에 다시 이어서 보기 쉬운 기능을 추가합니다.
- 외부 도서 Open API가 비정상 응답을 줄 때, 지금보다 더 이해하기 쉬운 안내와 복구 방법을 제공합니다.
