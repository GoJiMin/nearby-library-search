# Phase 1. App 레이어 표준화 Task

## 1. 프로젝트 기반 전환
- [x] `react-router`, `@tanstack/react-query`, `tailwindcss` 관련 의존성을 추가한다.
- [x] TypeScript 전환에 필요한 의존성과 설정 파일을 추가한다.
- [x] Vite 설정을 TypeScript 기반으로 정리한다.
- [x] 경로 별칭과 기본 import 규칙을 정리한다.
- [x] `vitest`와 `react testing library` 기반 테스트 환경을 추가한다.

## 2. FSD 디렉터리 구조 정리
- [x] `src/app`, `src/shared`, `src/entities`, `src/features` 디렉터리를 생성한다.
- [x] 각 레이어의 기본 하위 구조를 정의한다.
- [x] 슬라이스 외부 공개를 위한 `index.ts` 기준 구조를 잡는다.
- [x] 기본 Vite 샘플 구조와 충돌하는 기존 파일 배치를 정리한다.
- [x] 라우트 화면은 `pages` 레이어 슬라이스에서 관리하도록 구조를 정리한다.
- [x] 페이지 슬라이스는 `ui` 세그먼트와 슬라이스 단위 `index.ts`를 사용하도록 정리한다.

## 3. 앱 엔트리와 Provider 구성
- [x] 앱 시작점을 `src/main.tsx`로 전환한다.
- [x] `StrictMode` 기준 최상위 렌더링 구조를 정리한다.
- [x] 앱 전역 Provider 조합용 엔트리 컴포넌트를 구성한다.
- [x] `QueryClient` 생성 위치와 기본 옵션을 정의한다.
- [x] Query Provider를 앱 전역에 연결한다.

## 4. Router 구성
- [x] `react-router` 기반 라우터 생성 파일을 작성한다.
- [x] 공통 레이아웃을 기준으로 `/` 단일 라우트를 구성한다.
- [x] 홈 진입 화면용 최소 페이지 셸을 구성한다.
- [x] 잘못된 경로 대응을 위한 최소 fallback 처리 기준을 마련한다.

## 5. 스타일 시스템 구성
- [x] Tailwind CSS를 Vite 프로젝트에 연결한다.
- [ ] 전역 스타일 진입점을 `src/app/styles/globals.css`로 구성한다.
- [ ] 베이스 스타일과 리셋 규칙을 적용한다.
- [ ] 최소 디자인 토큰을 CSS 변수로 정의한다.
- [ ] 앱 레이아웃에 사용할 기본 타이포그래피와 간격 기준을 반영한다.

## 6. 공통 레이아웃과 화면 상태 기준
- [ ] 공통 App Layout 컴포넌트를 작성한다.
- [ ] 페이지 최대 너비, 여백, 배경, 메인 콘텐츠 영역 기준을 반영한다.
- [ ] 공통 헤더 또는 서비스 타이틀 영역을 정의한다.
- [ ] 앱 초기 로딩 폴백 UI 기준을 마련한다.
- [ ] 앱 전역 최소 에러 화면 구조를 정의한다.
- [ ] 빈 상태 메시지의 기본 표현 기준을 정리한다.

## 7. 환경변수와 외부 SDK 진입 규칙
- [ ] `VITE_` 접두사 기준의 env 사용 원칙을 문서와 코드에 반영한다.
- [ ] env 접근용 단일 진입 유틸 또는 설정 모듈을 만든다.
- [ ] Kakao Map과 Open API 설정값을 확장 가능한 구조로 정리한다.
- [ ] 컴포넌트 내부의 직접적인 `import.meta.env` 사용을 제한하는 기준을 정리한다.

## 8. 기존 템플릿 정리
- [ ] 기본 Vite 데모 UI와 불필요한 예제 자산을 제거한다.
- [ ] 새 앱 셸 기준으로 초기 렌더링 화면을 교체한다.
- [ ] 이후 Phase 2~4 작업에 불필요한 샘플 코드가 남지 않도록 정리한다.

- [ ] Tailwind 유틸리티 스타일이 정상 적용되는지 확인한다.
