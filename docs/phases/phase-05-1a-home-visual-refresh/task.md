# Phase 5-1A. 홈 메인 화면 스크린샷 정합 리디자인 Task

- [x] `stitch_design_specification_implementation.zip`의 `screen.png`, `code.html`, `DESIGN.md`를 홈 리디자인의 참조 자료로 고정한다.
- [x] 스크린샷에서 따라갈 비율, 위계, 강조 방식과 프로젝트 문맥상 바꿔야 할 요소를 구분해 정리한다.
- [x] 홈 리디자인의 참조 우선순위를 `screen.png` → `code.html` → `DESIGN.md` 기준으로 고정한다.

## 2. 홈 히어로 카피와 구조 재설계

- [x] `HomePage`의 메인 제목을 스크린샷처럼 큰 에디토리얼 2행 구조로 재설계한다.
- [x] 보조 문단을 중앙 정렬, 제한된 폭, 짧은 카피 밀도로 정리한다.
- [x] 홈 화면의 전체 정렬과 최대 너비를 중앙 집중형 히어로 레이아웃으로 조정한다.

## 3. 검색 슬랩 구조 재구성

- [x] `BookSearchStart` 외곽을 큰 카드 패널이 아니라 떠 있는 단일 검색 슬랩 구조로 재구성한다.
- [x] 세그먼트 탭을 슬랩 상단의 작은 전환 레일로 배치한다.
- [x] 입력, 검색 아이콘, CTA를 한 줄 검색 행으로 재배치해 스크린샷과 비슷한 읽기 흐름을 만든다.
- [x] 검색 CTA를 짧고 강한 액션 블록 비율로 정리한다.

## 4. support 영역 시각 강등

- [x] 내부 섹션 제목이 히어로 집중도를 깨면 제거하거나 크게 약화한다.
- [x] 안내 문단과 예시 검색어를 검색 슬랩 아래의 보조 영역으로 재배치한다.
- [x] 예시 검색어를 카드형 섹션이 아닌 작은 칩/버튼 수준으로 정리한다.
- [x] support 영역이 검색 행보다 낮은 시각 우선순위를 가지도록 조정한다.

## 5. 표면 계층, 타이포, 강조 스타일 정밀 조정

- [x] 두꺼운 선과 카드 중첩 인상을 줄이고, 톤 차이와 부드러운 깊이감으로 구획을 재정리한다.
- [x] 제목, 본문, 입력, 탭, CTA의 타이포 비율을 스크린샷에 가깝게 재조정한다.
- [x] 블루 강조 구간, CTA 깊이감, 약한 유령 테두리 같은 핵심 시각 요소를 정제한다.
- [x] 홈 화면이 현재보다 더 조용하지만 더 또렷한 에디토리얼 인상을 갖도록 마감한다.

## 6. 반응형, 다크모드, 접근성 보정

- [x] 데스크톱에서 중앙 집중형 히어로 비율이 유지되도록 조정한다.
- [x] 모바일에서 검색 슬랩이 첫 fold 안에서 보이도록 제목 크기와 간격을 보정한다.
- [x] 다크모드에서도 표면 위계와 검색 슬랩 집중도를 유지하도록 보정한다.
- [x] 기존 tab semantics, 입력 label, helper text, 키보드 이동 접근성이 유지되는지 확인한다.

## Important Changes

- 이번 phase는 기능 추가가 아니라 홈 메인 화면의 시각 구조를 스크린샷에 가깝게 재구성하는 작업이다.
- `HomePage`와 `BookSearchStart`의 public 역할은 유지하고, 히어로 레이아웃과 검색 블록의 시각 인상만 크게 바꾼다.
- 테스트는 기존 기능 신뢰를 유지하면서 홈 route의 새로운 시각 구조를 함께 검증하는 방향으로 갱신한다.

## Test Plan

- 홈 route에서 큰 히어로 제목, 보조 문단, 검색 슬랩이 렌더링된다.
- 세그먼트 탭, 입력, 예시 검색어, CTA, Enter 제출이 기존처럼 동작한다.
- 홈 route에서 스크린샷과 유사한 중앙 정렬 히어로 구조가 유지된다.
- 데스크톱과 모바일 모두에서 검색 슬랩이 첫 행동으로 읽힌다.
- `test:run`, `lint:web`, `typecheck:web`, `build:web`를 통과한다.

## Assumptions

- `Phase 5-1` 완료 상태는 유지하고, 이번 작업은 후속 리디자인 phase로 분리한다.
- 스크린샷은 시각 정합의 기준이며, 영문 카피와 브랜드명은 그대로 사용하지 않는다.
- 홈 메인 화면의 인상을 우선적으로 맞추되, 기능 경계와 접근성 계약은 유지한다.
